# /CONTRIBUTING.md
# 说明：为贡献者提供提交流程、编码规范与验证清单。

## 提交流程
1. Fork 本仓库或在新分支开发功能，避免直接向 `main` 推送。
2. 使用约定式提交格式 `type(scope): message`，如 `web: 调整可访问性样式` 或 `mini: 修复触摸偏移`。
3. 在提交前确保本地通过测试脚本，并清理无关的调试文件。
4. 提交 Pull Request 时附上变更摘要、测试结果与截图（如涉及前端界面）。

## 编码规范
- JavaScript/TypeScript：保持逐行中文注释，方便团队成员快速理解逻辑。
- 不引入二进制产物（如 `.png`、`.mp4` 等），若需展示请提供外部链接。
- 尽量使用原生 API 与现有工具函数，避免引入新的第三方依赖。

## 本地验证
1. 运行无依赖测试脚本：
   ```bash
   node ./tests/smoke-core.mjs
   node ./tests/simulate-2048-runs.mjs
   ```
2. 手动打开 `/web/index.html` 验证键盘、触摸与高对比度模式。
3. 使用微信开发者工具导入 `/miniprogram/`，检查真机预览是否正常。

## 提交注意事项
- 若修改公共 API，请在 PR 描述中明确列出，并更新相关文档。
- 若新增配置或文件结构，请同步更新 `README.diff.md` 供后续追踪。
- 对于复杂问题，建议先在 Issue 中讨论再动手，避免重复劳动。
