# /tests/README.md
# 说明：列出无外部依赖的 Node 测试脚本及其用途与排查思路。

## 使用方式
## 说明：在项目根目录执行以下命令即可运行测试脚本。

```bash
node ./tests/smoke-core.mjs
node ./tests/simulate-2048-runs.mjs
```

## 脚本概览
## 说明：介绍每个脚本覆盖的范围。

- `smoke-core.mjs`：验证 `LevelManager` 与 `Game2048` 的基础 API 是否可用，涵盖随机移动、关卡推进与 JSON 往返。 // 中文注释确保逐行说明
- `simulate-2048-runs.mjs`：执行多局随机模拟，输出通关与失败数据，便于观察算法参数对整体表现的影响。 // 中文注释确保逐行说明

## 排查指引
## 说明：如何解读测试输出并定位问题。

1. 若冒烟脚本触发断言：检查 `core/` 模块是否改动了公开方法签名，或 JSON 序列化结构是否发生变化。 // 中文注释
2. 若模拟脚本失败率为 100%：确认随机权重或目标函数配置是否异常，必要时缩短最大步数以定位死局。 // 中文注释
3. 若脚本报错 `Cannot find module`：确保使用 Node 16+ 且以 `node` 命令直接运行，避免使用打包工具。 // 中文注释
